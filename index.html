Governance & Compliance Rules

System Interaction Guidelines

  • Rule G4 (Release Approval): All new features, designs, and content must be reviewed from the perspective of our key users (Consumers, Agents, Lawyers) before they can be seen by the public.
  • Rule G5 (Safe Dashboard Editing): When making updates to our command center or any critical dashboard files, the system must double-check that it isn't accidentally deleting existing content. If an update appears to wipe out a large portion of a file, the system must stop immediately and ask for CEO approval.
  • Rule G6 (Firebase Safety Guard): No automated system can reset or initialize hosting servers in a folder that already contains our working websites, ensuring our live platforms are never accidentally overwritten.
  • Rule G7 (System Security Guard): Automated systems are strictly forbidden from scanning personal computer directories that cause the operating system to freeze or demand manual security clicks.

Live Quality Rules & Checking Guidelines

All updates must pass the strict 2-Level QA Protocol before CEO review.

  • Level 1 - Visual QA: Mobile responsiveness, alignment, box borders, and CSS grid constraints mapped against DHV Premium Aesthetics.
  • Level 2 - Functional QA: Core mechanics must be demonstrated without server crashes (No API 400 errors or uncaught JS bugs).
  • Rule: Agent MUST NOT ask for CEO approval to fix their own failure. Automatically rollback or self-repair.
  • Rule: The output UI operates as a pristine, zero-trust Dashboard for VC demonstration. Everything must load smoothly.
